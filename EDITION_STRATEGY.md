# Edition Strategy

Private document. Do not publish.

## Model

This book follows a rolling open model.

- Edition N is written and structured privately.
- Edition N is not publicly released until Edition N+1 is published.
- Edition N may incorporate community improvements from Edition N-1.

## Release Process

When Edition N+1 is published:

1. Edition N is released publicly under CC BY 4.0.
2. The public GitHub repository is updated with Edition N content.
3. A git tag is created for historical traceability.
4. The README edition policy section is updated to reflect the new open edition.

## Revenue and Distribution

- Print and Kindle editions through Amazon (no exclusivity programs).
- The open version is always one edition behind.
- Revenue comes from the latest edition, workshops, and consulting.

## Contribution Handling

- The public repo accepts PRs under CC BY 4.0.
- Contributors agree their work may appear in future commercial editions with attribution.
- No CLA required unless scale grows significantly.

## Risk Acknowledgement

CC BY 4.0 allows anyone to repackage, print, sell, translate, or modify
the open edition. The moat is: author name, authority, the latest edition,
and the surrounding ecosystem (workshops, consulting, talks).
